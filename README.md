# CrewAI Sentinel Code Agent

AI-powered code security and quality analysis system built with CrewAI that can automatically detect issues, generate a remediation report, apply fixes to your Python source files, and push the changes back to GitHub.

---

## Architecture & Agents

This project implements a multi-agent "Sentinel" crew that automatically audits and fixes Python code.

**Security Agent (security_agent)**

- Role: Code Security Analyst
- Focus: OWASP-style issues in Python code, including SQL/command injection, path traversal, unsafe deserialization, weak authentication, hardcoded secrets/API keys, missing input validation, and dangerous API usage.
- Output: Structured Markdown list of security findings per file, with severity, explanation, affected snippet, and recommended fixes.

**Quality Agent (quality_agent)**

- Role: Code Quality Reviewer
- Focus: General code health issues such as missing type hints, dead/unused code, anti-patterns, poor modularization, readability problems, and performance pitfalls.
- Output: Markdown report of quality issues per file, with category, priority, explanation, snippet, and suggested improvements.

**Report Agent (report_agent)**

- Role: Technical Report Writer
- Focus: Takes the raw outputs from the Security and Quality agents and produces a single, professional Markdown report.
- Output: Consolidated audit written to `issues/summary_report.md` with:
  - Executive summary and totals by severity/priority
  - Issues grouped by file
  - Severity/priority breakdowns
  - Actionable, developer-friendly recommendations and overall health assessment

**Code Fixer (code_fixer)**

- Role: Secure Code Fixer
- Focus: Reads the vulnerability/quality report in its context and rewrites every affected Python file with secure, production-ready code, changing only what is necessary to fix the issues.
- Output: For each vulnerable file, emits a complete, patched file (plus a short summary) into `issues/fixed_files_report.md`, using a strict, machine-parsable format.

---

## Tools (including GitHub pusher)

**Read Source File Tool (`read_source_file`)**

- Purpose: Safely read the full contents of a Python source file from an absolute path so agents can analyze the exact version on disk.

**Write Source File Tool (`write_source_file`)**

- Purpose: Overwrite an existing Python file with the fixed content generated by the Code Fixer.
- Safety: Only writes to `.py` files and returns a clear success or error message.

**GitHub Push Tool (`github_push`)**

- Purpose: Automates committing and pushing all modified files to a GitHub repository once fixes are applied.
- Behavior:
  - Detects the Git repository root and always stages from there.
  - Updates the `origin` remote to use an authenticated HTTPS URL (token is never logged).
  - Stages all changes, creates a commit (allowing empty commits), and pushes to the configured branch.
- Required environment variables:
  - `GITHUB_TOKEN` – Personal access token with appropriate repo permissions.
  - `GITHUB_REPO` – `<owner>/<repo>` of the target repository.
  - `GITHUB_BRANCH` – Branch to push to (defaults to `main` if not set).
- Output: A success message with the target repo/branch, or a sanitized error if any git step fails.

## End-to-End Workflow

1. **Collect source files** – Python source code paths and contents are gathered and injected into the tasks as `{source_code}`, clearly labeled by file.
2. **Security analysis** – The Security Agent runs `analyze_security_task`, scans each file, and produces a detailed Markdown list of vulnerabilities with severities and suggested fixes.
3. **Quality analysis** – The Quality Agent runs `analyze_quality_task` on the same set of files, producing a complementary list of code quality findings and improvements.
4. **Report generation** – The Report Agent runs `generate_report_task`, merging both analyses into a single, navigable report written to `issues/summary_report.md`.
5. **Automated fixing** – The Code Fixer runs `fix_vulnerabilities`, consumes the report as context, and emits complete, fixed versions of every vulnerable file into `issues/fixed_files_report.md`.
6. **Apply patches to the repo** – A driver (or another agent) reads `fixed_files_report.md` and uses `write_source_file` to overwrite the corresponding `.py` files in the repository.
7. **Commit and push to GitHub** – The `github_push` tool is invoked with a commit message (e.g., `fix: auto security patch`), staging all changes, committing, and pushing to the configured GitHub repository and branch.

## Real-World Value

- Provides a repeatable pipeline to scan, report, fix, and ship secure, high-quality Python code.
- Integrates directly with GitHub, enabling fully or semi-automated remediation workflows where each run can open or update a branch with auto-generated security fixes.
